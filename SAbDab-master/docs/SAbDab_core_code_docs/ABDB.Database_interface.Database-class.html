Class Database

source code

object --+
         |
        Database

The database class.

This is the back-end parser and interface for SAbDab.

The "fetch" method is used to get an object that contains the details about a structure.

e.g. API useage >>> from ABDB import database # database is initialised when ABDB is called. >>> p = database.fetch("12e8") >>> p.get_species() 'MUS MUSCULUS'

deposition_plot(self, png='')

source code 

Generate a plot of the number of antibody structures in the PDB by year. This uses the the deposition date from PDB files.

A matplotlib plot is produced which can then be saved.

Note that the numbers per year seem to decline in the current and previous year. My guess is that this is because the PDB may hold back structures until checks are made - therefore the latest update may have deposition dates from the previous year (or two)

Requires: matplotlib, numpy

get_template(self, seq, region=[], strict=False, fallback=False, n=10, use_similarity=False)

source code 

Get the top n templates by sequence identity for your sequence.

Parameters:
  • seq - The sequence you wish to model. Use the chain break symbol "/" to submit heavy and light chains together. e.g QEIDLF/SDFMMSD
  • region - The region over which you wish to calculate sequence identity.
  • n - The number of templates that should be returned. Default is 10
  • strict - If True only templates that have 100% coverage over the region specified (or variable region) will be returned.
  • fallback - If abnum fails to number the sequence try to do so with muscle number.
  • use_similarity - If True, use sequence similarity instead of sequence identity. Sequence similarity calculated using blosum62
Returns:
A list of templates [ ((pdb, chain, model), identity), ....] or for h/l [ ((pdb, H, L, model), identity), ....] AND the numbered sequence that was submitted.

["vl","vh","framework","interface","cdrs","cdrh1","cdrh2","cdrh3","cdrl1","cdrl2","cdrl3"]

get_identities(self, sequence_dict, region=[], strict=False, use_similarity=False)

source code 

Get the identities of a numbered sequence to the variable regions in the database.

Parameters:
  • sequence_dict - A dictionary containing the numbered sequence position-aa dictionary. Chain type (H or L) as keys.
  • region - The region over which you wish to calculate sequence identity.
  • strict - If True only templates that have 100% coverage over the region will be given a sequence identity greater than 0.
  • use_similarity - If True, use sequence similarity instead of sequence identity. Sequence similarity calculated using blosum62

    @return A dictionary containing the identities to the database entries.

get_identical(self, sequence_dict, region=[], threshold=0.99, strict=False, use_similarity=False, get_fabs=False)

source code 

Get identical (or within a sequence identity threshold) structures to your sequence.

Parameters:
  • sequence_dict - A dictionary containing the numbered sequence as a position-aa dictionary. Chain type (H or L) as keys.
  • region - The region over which you wish to calculate sequence identity.
  • threshold - The threshold at which to call things the same (default is >99%). Changes to blosum62 score if use_similarity is true
  • strict - If True only templates that have 100% coverage over the region will be given a sequence identity greater than 0.
  • use_similarity - If True, use sequence similarity instead of sequence identity. Sequence similarity calculated using blosum62
  • get_fabs - Fetch these structures from the database? These fab objects become the values of the returned dictionary
