Change Log


Changes in ABDB.ABDB_updater.Generate_summary
  • get_imgt_fields changed to comply with new IMGT updating methods 020513
  • added affinity data.
  • added species for H L and ag
  • added delta_g and pmid (source of affinity data)
Changes in ABDB.ABDB_updater.IMGT
  • New fetch and parsing methods 020513. We now get all the genes we can from imgt (v, j, c)
  • New assignment methods added 150513. We can now assign v j c genes without relying on imgt webserver.
Changes in ABDB.ABDB_updater.Identify
  • identify_partial method made private i.e. _identify_partial JD 150513
  • Documentation added JD 150513
  • Permissions flags added for compatibility with configuration.
  • Made anarci the default algorithm: JD 140115
Changes in ABDB.ABDB_updater.Populate
  • Fv nomenclature changed to Fab 020513
  • output_imgt changed to output all gene annotations for the chain. 020513
  • populate_part changed so that it can use (soon to be) any method of the Populator
Changes in ABDB.ABDB_updater.Populate.Populator.output_annotation
  • Adding in storage of all numbering schemes (chothia, kabat, martin and imgt)
Changes in ABDB.AbPDB.ABchain
  • unnumbered list now contains the id's of the unnumbered residues. Unnumbered residues are now yielded by iterating over the list.
  • The child residues now have a chothia numbered flag (True if chothia numbered false otherwise).
  • non-amino acid HETATMs now included in the children. (and reference in unnumbered)
  • ABchain.antigen is now a list. Each member of the list is a bound antigen. This handles multi-chain antigens.
Changes in ABDB.AbPDB.AntibodyParser'
  • Added support for IMGT numbering.
Changes in ABDB.AbPDB.AntibodyStructure
  • Fv nomenclature changed to Fab 020513
Changes in ABDB.AbPDB.Fab
  • Fab.antigen is now a list. Each member of the list is a bound antigen. This handles multi-chain antigens
Changes in ABDB.AbPDB.Model.Model
  • __getitem__ changed so that single chains can be called as well as holder object from a model. e.g. s[0]["H"] and s[0]["HL"] gets the H chain and the HL fab respectively.
Changes in ABDB.AbPDB.Residue
  • chothia numbered flag added. defualt is false.
Changes in ABDB.AbPDB.Visualise
  • Modified the visualise function to take a list of entities as input and show them all in the same session. The session will be associated with the first entity in the list
Changes in ABDB.Annotate.annotate'
  • Added the new version of anarci. This overwrites the old anarci function which is available as anarci_old.
  • anarci must be intalled separately now as it will be distributed as a separate package.
Changes in ABDB.Annotate.annotate'.extract_sequence
  • Residues containing HETATOMs are skipped --> Residues containing HETATOMs are checked as an amino acid.

    Residues containing HETATOMs are checked to be amino acids and the single letter returned.

    This works provided the residues in the chain are in the correct order.

Changes in ABDB.Database_interface.Antigen_details._set_antigen_name
  • uniquify the names but preserve ordering
Changes in ABDB.Database_interface.Database._read_db_summary
  • Check to see whether the summary file found in the loaded database has all the fields expected by the interface. If not, the user is told and None is passed to the fields without information.
Changes in ABDB.Database_interface.Fab_details._set_CDR_sequences
  • Now using the new Accept object from regions module and compatibility with the different schemes
  • Added heuristics to do conversion of non-equivalent positions.
  • Now use the annotate regions function as a utility function from AB_Utils
Changes in ABDB.command_line
  • Changed to use Fab nomenclature instead of Fv 030513
Changes in ABDB.config
  • Incorporated the ability to automatically identify kabnum_wrapper.pl. Enter to accept defaults. Ack: SK
  • Incorporated SK's suggestions for global setups.
  • Added PDB mirror compatibility
